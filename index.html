Endcaps V2

What's all this, then?

The goal of endcaps V2 is pretty straightforward:

  • Establish a cleaner starting point
  • Add clearer namespacing to our elements
  • Include certain functionality that would otherwise require additional actions
  • (Hopefully) Lay the groundwork for a responsive future

This version of endcaps uses border-box in the CSS so each element never exceeds the width it's initially given and uses a more semantic unordered list to display each of the elements. Further, all animation is handled with CSS, which allows for hardware acceleration for smoother transitions where appropriate.

Feature Requests

Below are an unordered list of things that would make working with endcaps way easier. Some of them would be nice to have (or at least get on the radar), others seem like no-brainers.

  • Default text-based arrows that can easily be styled using font controls (as shown above)
  • Ability to add a unique ID or class—helpful for sites with multiple endcaps running on a single page
  • Indexed classes added to each endcap item (like with product grids)
  • Make image creatives optional
    • Add the ability to choose between images or text for creatives, as shown above
  • Scrollable on mobile devices. With the new structure, it should be easier to enable.
  • Nice to have: Combine horizontal and vertical endcaps into one op. Choose layout with radio buttons/checkbox.
  • Nice to have: Ability to reorder feed output. Some clients like have alt price before main price, brand below product name, etc. We can use absolute positioning, but reordering would be ideal.
  • Nice to have: Ability to Add to Cart right from within the endcap, though I know this is hugely complex
  • Polling on endcaps
  • Badging on endcaps
